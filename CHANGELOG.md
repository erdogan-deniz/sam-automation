# Changelog

Все значимые изменения проекта. Формат — по [semver](https://semver.org).

## [1.3.0]

### Steam CM авторизация (scan)

- Полностью переработан вход в Steam CM для аккаунтов на современном Steam auth.
  Раньше `scan` бесконечно переспрашивал «Неверный пароль» и не получал ни одной
  игры по CM, хотя те же креды входят в веб/клиент Steam.
- Корень: legacy `client.login(пароль)` (ClientLogon) Steam отвергает для таких
  аккаунтов с `InvalidPassword` даже на ВЕРНОМ пароле. Теперь при этом
  выполняется современный путь `IAuthenticationService` (RSA + 2FA → JWT) и
  логон в CM по токену.
- Исправлена обработка Steam Guard 2FA: код мобильного аутентификатора
  (`DeviceCode`) больше не игнорируется; push-подтверждение
  (`DeviceConfirmation`) не запрашивает код, а ждёт подтверждения в приложении.
- CM-логон по JWT собирается как штатный `client.login` (ожидание шифрования
  канала + `steamid` в заголовке) и предъявляет `refresh_token` SteamClient-
  scope — короткий `access_token` CM отвергал с `AccessDenied`.
- После успешного входа `refresh_token` кэшируется (отдельно от веб-кук) →
  следующий скан входит в CM без пароля и 2FA.
- Прерван бесконечный ре-промпт пароля: реально неверный пароль завершается
  быстро, а не виснет.

### Auth (общее)

- Кэш `refresh_token` больше не удаляется при транзиентной ошибке сети
  (таймаут/нет ответа) и при пустом деривированном токене — стирается только
  при явном протухании/отзыве. Раньше кратковременный сбой сети заставлял
  вводить пароль/2FA каждый запуск.

## [1.2.0]

### Playtime (boost)

- По умолчанию набивает время во **всех** играх из `all.txt` (вся библиотека),
  а не только в тех, что отдаёт Steam Web API. Игры без данных API
  (free/демо/лицензии) идлятся вслепую один раз.
- Известные игры снимаются из набивки **по реальному** `playtime_forever`
  (перепроверяется каждый прогон), а не автоматически после одного идла;
  не добравшие target — повторяются. Невидимые API игры пишутся в
  `playtime/done.txt` (resume). Не подключившиеся → `playtime/skip.txt`.
- Скорость: отдельный `playtime_concurrent_games` (не общий с картами) и
  настраиваемый `launch_stagger` — главные рычаги пропускной способности.
- Флаг `--reset` сбрасывает прогресс набивки (`playtime/done.txt`).

### Achievements / Cards (farm)

- Заработали флаги `--retry-errors` / `--reset` / `--no-resume` (был добавлен
  argparse) — кнопки GUI «Retry errors» / «Reset» больше не пустышки;
  `cards/farm.py` получил рабочий `--reset`.

## [1.1.1]

### Откат

- Полностью откатан каталог достижений из 1.1.0. Код возвращён к состоянию
  1.0.0. Причина: классификация `scan` по Store API могла пометить игру с
  достижениями как «без достижений» (Store отдаёт пустой/отсутствующий блок
  achievements для playtest/демо/регион-лока/снятых с продажи), после чего
  farm пропускал её навсегда — нарушалась гарантия «всё неясное доезжает до
  SAM». Будет переделано с разделением «догадка scan» и «факт SAM».

## ~~[1.1.0]~~ (откатано в 1.1.1)

### Achievements (scan)
- `scan` после сбора `all.txt` классифицирует игры по наличию достижений
  через Steam Store API и раскладывает их на `with.txt` (есть достижения)
  и `without.txt` (нет) — без запуска SAM.
- Уже классифицированные игры (а также `unlocked.txt` от farm) повторно не
  опрашиваются; приложения, которых нет в Store (DLC, удалённые, серверы),
  не записываются и перепроверяются при следующем скане.
- Результат пишется по мере получения — прерывание безопасно, следующий
  запуск продолжает с нерасклассифицированных.

## [1.0.0]

Первый формальный релиз: стабильные farm / boost / cards + git-flow.

### Achievements (farm)
- Retry + нажатие **Refresh** при незагрузке статистики — игра больше не
  теряется в `error` из-за временной заминки.
- Быстрый поиск кнопки Refresh в `_MainToolStrip` (0.2 с вместо 4.6 с).
- Короткая перепроверка после Refresh; ранний выход для битых игр с пустым
  статусом (~8 с вместо 20 с).
- Откидывание в `error`, если статистика так и не загрузилась (без вечного
  RETRY).

### Cards (farm)
- Запуск `SAM.Game.exe` по очереди с паузой — устранена гонка за Steam
  global user.
- Окно браузера для входа в Steam открывается на весь экран.

### Playtime (boost)
- Поэтапный запуск батча с паузой между стартами и между батчами.
- Детект и skip игр, не подключающихся к Steam (playtest и пр.).

### Общее
- Защита от параллельного запуска farm / boost / cards (run-lock).
- Логирование: единый паттерн логгера, хелперы `SEPARATOR`/`centered`,
  централизованный UTF-8.
- `ruff format` по всей кодовой базе, pre-commit хуки (ruff + базовые).
- Документация git-flow процесса (`docs/gitflow.md`).
