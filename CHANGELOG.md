- **Удалены `scripts/stats.py` и `app/stats.py`** (сводка по библиотеке

## [1.9.0]

### Возможности

- **`--retry-without` вместо `--no-resume`** (`scripts/achievements/farm.py`) —
  точечная перепроверка ТОЛЬКО игр без достижений: `without.txt` (SAM-терминал)

  Срез `--retry-without` самосокращается, `stats` не задваивает игру. Новый
  атомарный примитив `_remove_id` (`app/id_file.py`, пустой файл удаляет).

### Удаление

- **Убран CLI-флаг `-v/--verbose`** из `farm`/`cards/farm`/`boost` — он лишь

## [1.7.0]

### Playtime boost — надёжность и корректность (глубокий аудит, 42 агента)

- **Потеря игры (HIGH):** known-игра при разовом (часто транзиентном) провале
  подключения больше НЕ хоронится в `skip.txt` навсегда — skip только для
  unknown; known перепроверяет Steam API на следующем прогоне. Добавлены
  `clear_playtime_skip` и флаг `--retry-skips` для восстановления ранее
  отсеянных игр.
- **Свип процессов (HIGH):** любое исключение в `_boost_loop` (не только
  Ctrl+C) добивает активные + `kill_all_sam_games` через `finally` —
  `SAM.Game.exe` больше не сиротеют и не занимают global user.
- **Честный отчёт:** ✅-тост/Telegram только при полном прогоне без провалов;
  Ctrl+C, ошибка и наличие провалов → ⚠️ и честный текст (`_report_result`).
- **Пустой owned-games:** предупреждение при приватных Game details (все игры
  классифицируются unknown); `main` ловит `RuntimeError` из `_fetch_targets`.
- **Валидация конфига:** `playtime_idle_duration≥1`, `playtime_target_minutes≥1`,
  `launch_stagger≥0`; warning на не-список `exclude_ids`/`game_ids`.

### Run-lock и GUI

- **Run-lock:** берётся ДО деструктивного `--reset`/`--retry-skips` (`--list`
  теперь read-only). Формат `PID:create_time:name` отсекает PID-reuse, захват
  атомарен (`os.open O_EXCL`), release снимает ТОЛЬКО собственный лок.
- **Teardown:** второй Ctrl+C во время уборки не обрывает свип (bounded-retry).
- **GUI:** сохранение настроек мержит `config.yaml` — больше не затирает
  `playtime_concurrent_games`/`launch_stagger`/`playtime_target_minutes`/
  `telegram_*`/`game_ids`, которых нет в форме.

## [1.6.0]

### Фарм карточек (cards)

- **Fast-mode цикл фарма**: пачка идлится → все окна убиваются РАЗОМ (коллапс
  in-game в ноль) → пауза-flush → перечитка остатка → выжившие в очередь и
  релонч. Устойчивая пагинация badges с ретраями на страницу (дёрганый SSL не
  теряет игры), stall-guard. Честный финальный отчёт: сдача по cookie-ошибке и
  застревания НЕ пишут `done` и не дают success-тост (тост «прерван» / «с
  оговорками» / чисто).

### Уведомления (notify)

- **Telegram-уведомления** рядом с локальным Windows-toast (opt-in по
  `telegram_*` в конфиге, best-effort).

### GUI

- **Остановка скрипта убивает всё дерево процессов** через Win32 Job Object
  (`KILL_ON_JOB_CLOSE`) — Stop/Esc/закрытие окна больше не оставляют сирот
  `SAM.Game.exe` (аккаунт не застревает in-game). Закрытие окна GUI сперва
  останавливает запущенные скрипты.

### Набивка playtime (boost)

- Отсев провалившихся запусков по **выживанию процесса** за весь idle, а не по
  транзиентному окну ошибки `SAM.Game.exe` (гонка). Страховка от сирот при
  Ctrl+C во время старта батча.

### Устойчивость сети (hardening)

- Сетевые запросы (`card_checker._fetch_page`, `steam_api._api_get`,
  `validator._check_steam_api`, `sam_downloader`) ловили только urllib
  `HTTPError`/`URLError`. При обрыве тела ответа `http.client.IncompleteRead`
  и при 302-редиректе `RemoteDisconnected` (подклассы `OSError`/`HTTPException`,
  но НЕ `URLError`) проходили мимо `except` и роняли весь прогон сырым
  исключением. Теперь оборачиваются в `RuntimeError` → штатный ретрай /
  чистый `sys.exit(1)` / `[CONFIG ERROR]`. (#13, #14, #15)

### Конфиг и валидация

- Числовые границы в pre-flight валидации (`max_concurrent_games`,
  `playtime_concurrent_games` ∈ [1, 20]; `card_check_interval` ≥ 1) — руками
  правленный `0`/отрицательный конфиг больше не даёт тихий no-op/busy-loop.

### Инфраструктура и документация

- `CLAUDE.md` (durable-правила проекта), `SECURITY.md`, `docs/prompts/`
  (playbook'и farm/boost/scan + аудитор). CI/pre-commit hardening: пин `ruff`,
  release-gate (VERSION==CHANGELOG==тег), UTF-8 stdout в `check_version`.
  Удалён мёртвый модуль `card_store`.

## [1.3.0]

### Steam CM авторизация (scan)

- Полностью переработан вход в Steam CM для аккаунтов на современном Steam auth.
  Раньше `scan` бесконечно переспрашивал «Неверный пароль» и не получал ни одной
  игры по CM, хотя те же креды входят в веб/клиент Steam.
- Корень: legacy `client.login(пароль)` (ClientLogon) Steam отвергает для таких
  аккаунтов с `InvalidPassword` даже на ВЕРНОМ пароле. Теперь при этом
  выполняется современный путь `IAuthenticationService` (RSA + 2FA → JWT) и
  логон в CM по токену.
- Исправлена обработка Steam Guard 2FA: код мобильного аутентификатора
  (`DeviceCode`) больше не игнорируется; push-подтверждение
  (`DeviceConfirmation`) не запрашивает код, а ждёт подтверждения в приложении.
- CM-логон по JWT собирается как штатный `client.login` (ожидание шифрования
  канала + `steamid` в заголовке) и предъявляет `refresh_token` SteamClient-
  scope — короткий `access_token` CM отвергал с `AccessDenied`.
- После успешного входа `refresh_token` кэшируется (отдельно от веб-кук) →
  следующий скан входит в CM без пароля и 2FA.
- Прерван бесконечный ре-промпт пароля: реально неверный пароль завершается
  быстро, а не виснет.

### Auth (общее)

- Кэш `refresh_token` больше не удаляется при транзиентной ошибке сети
  (таймаут/нет ответа) и при пустом деривированном токене — стирается только
  при явном протухании/отзыве. Раньше кратковременный сбой сети заставлял
  вводить пароль/2FA каждый запуск.

## [1.2.0]

### Playtime (boost)

- По умолчанию набивает время во **всех** играх из `all.txt` (вся библиотека),
  а не только в тех, что отдаёт Steam Web API. Игры без данных API

- Известные игры снимаются из набивки **по реальному** `playtime_forever`
  (перепроверяется каждый прогон), а не автоматически после одного идла;
  не добравшие target — повторяются. Невидимые API игры пишутся в
  `playtime/done.txt` (resume). Не подключившиеся → `playtime/skip.txt`.
- Скорость: отдельный `playtime_concurrent_games` (не общий с картами) и
  настраиваемый `launch_stagger` — главные рычаги пропускной способности.
- Флаг `--reset` сбрасывает прогресс набивки (`playtime/done.txt`).

### Achievements / Cards (farm)

- Заработали флаги `--retry-errors` / `--reset` / `--no-resume` (был добавлен
  argparse) — кнопки GUI «Retry errors» / «Reset» больше не пустышки;
  `cards/farm.py` получил рабочий `--reset`.

## ~~[1.1.0]~~ (откатано в 1.1.1)

### Achievements (scan)
- `scan` после сбора `all.txt` классифицирует игры по наличию достижений
  через Steam Store API и раскладывает их на `with.txt` (есть достижения)
  и `without.txt` (нет) — без запуска SAM.
- Уже классифицированные игры (а также `unlocked.txt` от farm) повторно не
  опрашиваются; приложения, которых нет в Store (DLC, удалённые, серверы),
  не записываются и перепроверяются при следующем скане.
- Результат пишется по мере получения — прерывание безопасно, следующий
  запуск продолжает с нерасклассифицированных.

## [1.0.0]

Первый формальный релиз: стабильные farm / boost / cards + git-flow.

### Achievements (farm)
- Retry + нажатие **Refresh** при незагрузке статистики — игра больше не
  теряется в `error` из-за временной заминки.
- Быстрый поиск кнопки Refresh в `_MainToolStrip` (0.2 с вместо 4.6 с).
- Короткая перепроверка после Refresh; ранний выход для битых игр с пустым
  статусом (~8 с вместо 20 с).
- Откидывание в `error`, если статистика так и не загрузилась (без вечного
  RETRY).

### Cards (farm)
- Запуск `SAM.Game.exe` по очереди с паузой — устранена гонка за Steam
  global user.
- Окно браузера для входа в Steam открывается на весь экран.

### Playtime (boost)
- Поэтапный запуск батча с паузой между стартами и между батчами.
- Детект и skip игр, не подключающихся к Steam (playtest и пр.).

### Общее
- Защита от параллельного запуска farm / boost / cards (run-lock).
- Логирование: единый паттерн логгера, хелперы `SEPARATOR`/`centered`,
  централизованный UTF-8.
- `ruff format` по всей кодовой базе, pre-commit хуки (ruff + базовые).
- Документация git-flow процесса (`docs/gitflow.md`).
